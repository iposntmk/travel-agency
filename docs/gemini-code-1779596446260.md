# TC Travel Vietnam  - Tài liệu Hướng dẫn Lập trình & Kiến trúc Hệ thống Toàn diện

**Tên dự án:** TC Travel Vietnam
**Stack kỹ thuật:** Next.js 15 (App Router), Payload CMS (Co-located), TypeScript (Strict Mode), Tailwind CSS, shadcn/ui, pnpm, Clerk Auth, Neon Postgres (Vùng Singapore).

Tài liệu này là bộ khung kiến trúc tối cao, bắt buộc và không thể thương lượng dành cho tất cả các công cụ trợ lý lập trình AI (Claude Code, DeepSeek, Cursor) và lập trình viên. Mọi dòng mã nguồn sinh ra phải tuân thủ nghiêm ngặt các giới hạn, kiến trúc phân tầng, và sơ đồ thư mục dưới đây.

---

## 1. Sơ đồ Cây Thư mục Tổng quan (Directory Tree)

Mọi tệp tin mới được khởi tạo bắt buộc phải nằm đúng vị trí trong cấu trúc phân tầng dưới đây để đảm bảo tính Modular, DRY, và bảo mật:

```text
travel-agency/
├── .github/                     # Cấu hình GitHub Actions, CI/CD Workflows
├── public/                      # Tài nguyên tĩnh công khai (Favicon, Logo, Fonts cố định)
├── src/                         # Thư mục mã nguồn chính của ứng dụng
│   ├── app/                     # Next.js 15 App Router (Tầng Giao diện & Định tuyến)
│   │   ├── (app)/               # Nhóm định tuyến cho giao diện chính của khách hàng (B2C)
│   │   │   ├── layout.tsx       # Bố cục giao diện khách hàng (Root Layout của Customer App)
│   │   │   ├── page.tsx         # Trang chủ  TC Travel Vietnam
│   │   │   ├── tours/           # Trang danh sách tour và chi tiết tour
│   │   │   │   ├── page.tsx     # Bộ lọc tour theo mùa & thị trường (RSC)
│   │   │   │   └── [slug]/      # Trang chi tiết tour (ví dụ: /tours/hue-walking-tour)
│   │   │   │       └── page.tsx
│   │   │   ├── destinations/    # Trang điểm đến (Huế, Hội An, Đà Nẵng...)
│   │   │   ├── wellness/        # Trang dịch vụ add-on bổ sung (Spa, Nha khoa, Beauty)
│   │   │   └── confirmation/    # Trang xác nhận sau khi submit Booking Inquiry thành công
│   │   ├── (payload)/           # Nhóm định tuyến dành riêng cho Payload CMS Admin & API Panel
│   │   │   ├── admin/           # Bảng quản trị hệ thống nội bộ của Agency (/admin)
│   │   │   └── api/             # API endpoints nội bộ và Webhooks từ bên thứ ba (/api)
│   │   └── actions/             # Tầng Đột biến Dữ liệu (Next.js Server Actions)
│   │       ├── bookings.ts      # Xử lý gửi biểu mẫu đặt tour, validate Zod và chặn spam
│   │       ├── comments.ts      # Xử lý thêm, duyệt bình luận của khách
│   │       └── reviews.ts       # Xử lý lưu trữ review sau khi khách hoàn thành tour
│   │
│   ├── collections/             # Cấu hình Thực thể Dữ liệu (Payload CMS Collections)
│   │   ├── users/               # Cấu hình tài khoản Admin/Staff và khách hàng link với Clerk
│   │   ├── tours/               # Cấu hình dữ liệu tour (Bậc giá, lịch trình, trạng thái vận hành)
│   │   ├── destinations/        # Cấu hình thông tin điểm đến vùng miền
│   │   ├── bookings/            # Cấu hình đơn đặt tour (Trạng thái Enum, thông tin liên lạc)
│   │   │   ├── hooks/           # Vòng đời dữ liệu cục bộ (Lifecycle Hooks)
│   │   │   │   ├── validateStatus.ts # Hook kiểm tra tính hợp lệ khi chuyển đổi trạng thái đơn
│   │   │   │   └── notifySales.ts    # Hook tự động gửi thông báo WhatsApp/Email cho Sales team
│   │   │   └── config.ts        # File định nghĩa trường dữ liệu collection chính
│   │   ├── partners/            # Quản lý đối tác outsource tour và cơ sở dịch vụ add-on bổ sung
│   │   └── media/               # Quản lý tệp đa phương tiện, tích hợp siêu dữ liệu Cloudflare R2
│   │
│   ├── components/              # Các thành phần giao diện UI/UX tái sử dụng toàn cục
│   │   ├── ui/                  # Linh kiện nguyên bản từ thư viện shadcn/ui (Button, Dialog...)
│   │   ├── widgets/             # Các widget phức hợp tích hợp chức năng nâng cao
│   │   │   ├── OTAWidget.tsx    # Thành phần nhúng liên kết affiliate (Civitatis, GetYourGuide)
│   │   │   └── SocialShare.tsx  # Thành phần chia sẻ bài viết/tour lên mạng xã hội có Click-to-load
│   │   ├── forms/               # Các form nhập liệu đồng bộ hệ thống
│   │   │   └── InquiryForm.tsx  # Biểu mẫu đăng ký đặt tour tích hợp React Hook Form + Zod
│   │   └── layout/              # Thành phần bố cục dùng chung (Navbar, Footer, CookieConsent)
│   │
│   ├── lib/                     # Cấu hình công cụ lõi, SDK kết nối bên thứ ba
│   │   ├── payload.ts           # Khởi tạo Payload Local API Client đảm bảo an toàn mã nguồn
│   │   ├── clerk.ts             # Khởi tạo client quản lý định danh người dùng qua Clerk
│   │   └── queue.ts             # SDK đẩy tác vụ xử lý ngầm qua Upstash / QStash
│   │
│   ├── services/                # Tầng Logic Nghiệp vụ Độc lập (Pure Business Services)
│   │   ├── pricing.ts           # Hàm xử lý tính toán giá tour theo bậc số lượng khách (pricingTiers)
│   │   └── consolidation.ts     # Hàm đánh giá trạng thái tour dựa trên chỉ số (currentPax vs minPax)
│   │
│   ├── schemas/                 # Thư mục chứa cấu trúc xác thực dữ liệu dùng chung (Zod Schemas)
│   │   ├── booking.schema.ts    # Schema dùng chung cho cả Inquiry Form (Client) và Server Action
│   │   └── partner.schema.ts    # Schema xác thực thông tin đối tác cung cấp dịch vụ bổ sung
│   │
│   └── styles/                  # Cấu hình CSS hệ thống
│       └── globals.css          # Điểm nhúng chính của Tailwind CSS và biến giao diện phong cách Huế
│
│   ├── payload.config.ts        # File cấu hình trung tâm cốt lõi của Payload CMS
│
├── tests/                       # Thư mục chứa các bài kiểm thử tự động (Automated Testing Matrix)
│   ├── actions/                 # Kiểm thử tích hợp Next.js Server Actions
│   │   └── booking-submit.test.ts
│   ├── schemas/                 # Kiểm thử độ chặt chẽ của các Schema Zod
│   │   └── validation.test.ts
│   └── collections/             # Kiểm thử quy tắc bảo mật Access Control của CMS
│       └── access-control.test.ts
│
├── .env.example                 # Bản mẫu định dạng các biến môi trường (Không chứa thông tin mật)
├── next.config.ts               # Cấu hình vận hành tối ưu của Next.js 15
├── tailwind.config.ts           # Định nghĩa palette màu chủ đạo (Vàng cam, Đỏ cờ, Xanh dương)
├── tsconfig.json                # Ràng buộc cấu hình biên dịch TypeScript Strict Mode
└── pnpm-lock.yaml               # Tệp khóa phiên bản thư viện duy nhất của dự án (Cấm npm/yarn)
